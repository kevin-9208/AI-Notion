写 AI 提示词（Prompt），本质上不是“写一句话给 AI”，而是在做一种**自然语言编程（Natural Language Programming）**。

你可以把模型理解成一个：

* **知识压缩器**（知道很多）
* **概率生成器**（猜你想要什么）
* **上下文解释器**（非常依赖你给的信息）

所以提示词质量，直接决定输出质量。

---

# 一、先理解：为什么提示词重要？

同样一个需求：

用户A：

> 写一个登录页面

用户B：

> 用 Vue3 + TailwindCSS 写一个响应式登录页，包含邮箱、密码、记住我、Google OAuth 按钮，深色模式支持，组件化拆分，适合 Supabase Auth 集成。

结果差距会非常大。

因为 AI 最怕：

* 模糊目标
* 缺上下文
* 缺约束
* 缺标准

提示词的作用就是：

把你的脑内模糊意图，压缩成 AI 可执行指令。

这就是核心。

---

# 二、提示词的底层结构（万能公式）

一个高质量 Prompt：

> 任务 + 上下文 + 约束 + 输出格式 + 标准

公式：

```text
[Role]
你是谁

[Task]
你要做什么

[Context]
背景信息是什么

[Constraints]
限制条件是什么

[Output Format]
输出格式是什么

[Quality Standard]
做到什么程度算合格
```

例子：

```text
你是一个资深 Vue3 工程师。

帮我写一个旅游规划系统的 Trips CRUD 模块。

技术栈：
- Vue3
- Pinia
- Supabase
- TailwindCSS

要求：
- Composition API
- TypeScript
- 可维护
- 模块化
- 错误处理完整

输出：
1. store
2. composable
3. page
4. component

代码必须是生产级。
```

这个结构非常稳定。

# 三、Prompt 的六大核心维度

---

# 1. Role（角色设定）

模型会根据角色切换行为模式。

比如：

开发：

```text
你是一个高级全栈工程师
```

产品：

```text
你是一个资深产品经理
```

设计：

```text
你是一个UI/UX设计专家
```

写作：

```text
你是一个专业科幻小说作家
```

心理：

```text
你是荣格心理学研究者
```

作用：

角色 = 决定视角

比如：

“解释数据库索引”

普通：

> 解释数据库索引

专家：

> 你是数据库内核工程师，深入解释 B+ Tree 索引实现原理

深度完全不同。

---

# 2. Task（任务清晰）

坏：

```text
帮我优化代码
```

好：

```text
帮我找出这个 Vue3 组件中的性能瓶颈，并优化渲染逻辑。
```

任务要：

* 明确动作
* 明确对象
* 明确目标

公式：

```text
动词 + 对象 + 目标
```

例：

* 分析这个项目架构
* 重构这段代码
* 优化 SQL 查询
* 设计数据库 schema
* 写测试用例

---

# 3. Context（上下文）

这个是高手和普通用户最大差距。

AI 非常吃上下文。

比如：

坏：

```text
写个后台
```

好：

```text
做一个旅游规划 SaaS 后台：

用户：
自由行游客

功能：
- 行程管理
- 地图标记
- 预算管理
- 协作分享

技术：
Vue3 + Supabase

目前已完成：
Auth、Trips 模块
现在做 Budget 模块
```

这会极大提高连续性。

---

# 4. Constraints（约束）

没有约束，AI 会自由发挥。

有时候自由发挥 = 偏题。

例：

```text
限制：
- 不要使用第三方状态库
- 不要用 Options API
- 必须 TypeScript
- 必须 SSR 兼容
- 优先考虑性能
```

常见约束：

技术约束：

* 用 React/Vue/Svelte
* 用 Python/Go/Rust

风格约束：

* 极简
* 工业风
* 苹果风

性能约束：

* O(n)
* 避免重复渲染

业务约束：

* 面向新手用户
* 支持多人协作

---

# 5. Output Format（输出格式）

这是被严重低估的部分。

比如：

坏：

```text
给我方案
```

好：

```text
按以下格式输出：

1. 问题分析
2. 架构设计
3. 数据库设计
4. API设计
5. 前端实现
6. 风险点
```

代码：

```text
输出完整目录结构
```

学习：

```text
用表格输出
```

比较：

```text
按优缺点对比
```

这会让结果稳定很多。

---

# 6. Quality Standard（质量标准）

非常强。

比如：

```text
不要给 demo 级代码，要给生产级代码。
```

或者：

```text
从架构层深入，不要停留在表面。
```

或者：

```text
假设读者是中级开发者。
```

或者：

```text
像源码作者一样解释。
```

质量标准决定深度。

# 四、高阶技巧（高手区）

---

# 技巧1：Few-shot（示例驱动）

你给例子，AI 学你的风格。

比如：

```text
按照这个风格继续：

输入：
用户登录失败

输出：
原因分析：
1...
2...

解决方案：
1...
2...
```

这叫示例学习。

很强。

---

# 技巧2：Chain of Thought（链式思考）

让 AI 分步骤思考。

坏：

```text
帮我设计系统
```

好：

```text
按步骤分析：

1. 明确需求
2. 找出核心实体
3. 设计数据库
4. 设计 API
5. 设计前端状态流
```

复杂任务提升巨大。

---

# 技巧3：Self-critique（自检）

非常强。

```text
完成后请检查：

- 是否有性能问题
- 是否有安全问题
- 是否有边界条件遗漏
```

代码质量会明显提高。

---

# 技巧4：Iterative Prompting（迭代提示）

高手基本不用“一发入魂”。

流程：

第一轮：

```text
先给整体架构
```

第二轮：

```text
展开数据库设计
```

第三轮：

```text
展开 Auth 模块
```

第四轮：

```text
展开错误处理
```

这才是正确玩法。

# 五、不同领域怎么写

---

# 编程

模板：

```text
你是资深XXX工程师，
帮我实现XXX，
技术栈XXX，
要求XXX，
输出完整代码+解释+边界处理。
```

---

# UI设计

模板：

```text
你是高级UI设计师，
设计一个XXX页面，
风格XXX，
用户群体XXX，
重点考虑XXX。
```

---

# 学习

模板：

```text
你是这个领域专家，
从基础到高级解释XXX，
用类比+案例+图景思维。
```

---

# 写作

模板：

```text
你是专业小说作家，
写一个XXX故事，
风格类似XXX，
节奏偏XXX，
重点塑造人物心理。
```

---

# 商业分析

模板：

```text
你是商业战略分析师，
分析XXX行业，
从市场、竞争、壁垒、风险展开。
```


---

最后给你一个开发者万能模板：

```text
你是资深全栈架构师。

我要做一个 [项目名]。

技术栈：
[技术栈]

目标：
[目标]

当前进度：
[当前进度]

请按以下步骤输出：
1. 需求分析
2. 架构设计
3. 数据模型
4. API设计
5. 前端状态管理
6. 权限设计
7. 性能优化
8. 风险点

要求：
- 生产级
- 可扩展
- 可维护
- 安全性完整
- 不要 demo 级方案
```

这套模板，基本能覆盖你 70% 的开发场景。
